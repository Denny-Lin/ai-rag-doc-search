# AI RAG Doc Search

An end-to-end AI-powered document search system built using Retrieval-Augmented Generation (RAG).  
Users can upload documents, perform semantic search, and receive accurate answers generated by large language models.

Live Demo: https://ai-rag-doc-search.vercel.app  
Backend API: https://ai-rag-doc-search.onrender.com/docs

---

## Overview

This project implements a full RAG pipeline that enables question answering over user-uploaded documents.  
It combines vector search with LLM inference to retrieve relevant context and generate grounded responses.

The system is designed as a full-stack application with a modern frontend and a scalable backend architecture.

## How It Works

### Document Processing

1. Upload PDF document  
2. Extract text content  
3. Split text into chunks  
4. Generate embeddings  
5. Store vectors in database (ChromaDB)

### Question Answering

1. Convert user query into embedding  
2. Perform vector similarity search  
3. Retrieve relevant chunks  
4. Send context to LLM (Groq - Llama 3)  
5. Generate final answer

## Getting Started

### 1. Clone Repository

```
git clone https://github.com/your-username/ai-rag-doc-search.git
cd ai-rag-doc-search
```

---

### 2. Backend Setup

```
cd backend
python3.11 -m venv venv
source venv/bin/activate
pip install -r requirements.txt
```

Run server:

```
uvicorn app.main:app --reload
```

---

### 3. Frontend Setup

```
cd frontend
npm install
npm run dev
```

## Current Limitations

### 1. Cold Start (Render)

- Backend becomes inactive when idle  
- First request latency: 20–60 seconds  

### 2. High Memory Usage

- Local embedding model consumes significant memory  
- Can exceed free tier limits on Render  

### 3. Slow Embedding Pipeline

- CPU-bound embedding generation  
- Not suitable for large-scale production  

### 4. No Persistent Storage

- Files stored locally in `/uploads`  
- Data lost after server restarts  

### 5. No Streaming Responses

- Responses returned after full generation  
- No real-time token streaming
